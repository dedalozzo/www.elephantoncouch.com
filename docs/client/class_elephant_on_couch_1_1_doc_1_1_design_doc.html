Detailed Description

A design document is a special CouchDB document where views, updates, rewrites and many others handlers are stored.

See also
http://guide.couchdb.org/editions/1/en/design.html

Adds a special handler to the design document.

Returns true if your document class already defines his type internally, false otherwise.

Sometime happens you have two classes with the same name but located under different namespaces. In case, you should provide a type yourself for at least one of these classes, to avoid Couch::saveDoc() using the same type for both. Default implementation should return false.

Removes the handler.

Some handlers belong to a section. For example a view ViewHandler belongs to the 'views' section. To specify the appropriate section name, you should use the static method getSection available for every handler implementation.
